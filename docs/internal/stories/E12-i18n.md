# E12 — Internationalization (French + English) — Detailed stories

> Epic goal: make Keepou **bilingual (French + English)** and let each member
> **choose their language** from the account menu (right under « Modifier mon
> nom »). French stays the product's first language (design/claude.md); English
> is an added locale, not a replacement — the frozen French wording of
> `HANDOFF.md` §7 is preserved verbatim as the reference locale.
>
> Estimation convention: **S** (≤ ½ day), **M** (1–2 days), **L** (3+ days).

**Reference docs.** `design/HANDOFF.md` §7 (the frozen French copy — the FR
locale) and §8 (« i18n: current copy in French; centralize the strings to ease a
later translation » — E8 already centralized them in one module, which this epic
turns into a locale system). `docs/ARCHITECTURE.md` §8 (auth / `User` model,
where the preference is stored). **Depends on** E8 (copy centralization) and E2
(`User` model + `PATCH /api/auth/me`, extended in E11).

**Key decision (validated).** The language preference is stored **server-side**
on the `User` (`language` column, default `FR`) so it follows the member across
devices, with a **localStorage mirror** so the SPA renders in the right language
before `/auth/me` resolves and on the pre-login screens. The server is
authoritative once logged in: the front adopts `User.language` on session load;
switching from the menu applies instantly, then persists via `PATCH /api/auth/me`.

## E12-S1 — Server-stored language preference · S

**Goal.** Persist each member's UI language so it follows them across devices.

**Tasks**
- `models.py`: `Language(StrEnum)` (`FR` / `EN`) + `User.language: Language = FR`.
- Migration `c5e2a1b9d3f4` — add `language` NOT NULL, `server_default='FR'` (existing
  members keep the francophone-first UI).
- `schemas.py`: `UserOut.language`; `ProfilePatch` — `display_name` **and**
  `language` both optional (each caller patches one), so `update_me` applies only
  the provided fields (empty body = no-op).

**Acceptance**
- [x] A new account defaults to `language == FR`.
- [x] `PATCH /api/auth/me {language: "EN"}` updates it and leaves the display name
      untouched; an unknown language is `422`.

---

## E12-S2 — Front i18n core · M

**Goal.** A single source of locale for the whole tree, flash-free at boot.

**Tasks**
- `web/src/i18n/fr.ts` — the reference dictionary (the former `lib/copy.ts` moved
  here, grouped by screen); `export type Copy = typeof fr`.
- `web/src/i18n/en.ts` — `export const en: Copy = {…}`; the `Copy` type makes the
  compiler reject any missing key or mismatched function signature (locales can't
  drift out of shape).
- `web/src/i18n/I18nProvider.tsx` — owns the active locale (localStorage →
  `navigator.language` → default FR), writes `document.documentElement.lang`,
  exposes `useI18n()` (the active dictionary) and `useLocale()` (locale + setter).
- `App.tsx` — `<I18nProvider>` wraps the tree (outside `AuthProvider`, so the
  auth context can adopt the server language via `useLocale`).

**Acceptance**
- [x] `useI18n()` returns the `*_COPY` groups for the active locale; switching
      locale re-renders every consumer.
- [x] `<html lang>` reflects the active locale.

---

## E12-S3 — Migrate consumers to the hook · M

**Goal.** No component reads a hardcoded/monolingual string anymore.

**Tasks**
- Replace `import { X_COPY } from '…/lib/copy'` with `const { X_COPY } = useI18n()`
  in every screen/component (≈ 34 files).
- `lib/colors.ts`: drop the baked-in swatch `label`; `ColorPicker` / `Composer`
  resolve `COLOR_LABELS[color]` from the hook at render.
- `ConfirmDialog`: copy defaults moved from module-scope constants into the body
  (a hook can't run in a default parameter value).
- Delete `web/src/lib/copy.ts`.

**Acceptance**
- [x] `tsc`, `eslint`, `vitest` green; no remaining import of `lib/copy`.

---

## E12-S4 — English translation · M

**Goal.** A complete, faithful English UI.

**Tasks**
- Translate every string in `en.ts`, mirroring the frozen French wording
  (HANDOFF §7). Language names stay endonyms (« Français » / « English ») in both
  locales.

**Acceptance**
- [x] Switching to English translates the whole UI (auth, board, editor, history,
      admin, import, agent-access); no French string leaks through.

---

## E12-S5 — Language switcher in the menu · S

**Goal.** Let the member pick their language where they'd expect it.

**Tasks**
- A compact FR/EN segmented control in the account menu, directly under
  « Modifier mon nom »; the menu stays open so the change is visible at once.
- `AuthContext.changeLanguage(locale)`: apply locally, then persist via
  `PATCH /me` when authenticated; adopt `User.language` on hydrate + sign-in.

**Acceptance**
- [x] Picking a language switches the UI immediately and persists (a fresh login
      on another device comes back in that language).
- [x] A failed persist keeps the local switch (offline-safe).

---

## E12-S6 — Localized timestamps · S

**Goal.** No French dates on an English board / history.

**Tasks**
- `lib/time.ts`: thread the active locale through the relative (« il y a 2 h » /
  « 2 h ago ») and absolute (history) formatters; call sites pass the locale.

**Acceptance**
- [x] In English, card meta and history labels read in English.
